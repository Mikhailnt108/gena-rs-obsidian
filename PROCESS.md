PROCESS.md

Version: v1
Standard: Core Documents Standard v1

Этот документ фиксирует канонический процесс работы в связке: Project Code ↔ Agent CLI ↔ Project Obsidian.

Процесс обязателен для человека и агента. Отклонения считаются ошибкой процесса.


---

1. Роли и зоны ответственности

Project Code

исходный код

тесты

конфигурации

CI/CD

скрипты и docker


Назначение: исполнение и проверка.


---

Project Obsidian

идеи

решения

текущие состояния

история действий


Назначение: мышление и память.


---

Agent CLI

читает память

исполняет действия

синхронизирует результат


Агент не хранит состояние в себе.


---

2. Обязательные документы

В Obsidian

IDEA.md

PROCESS.md

NOW.md

DL.md

WORKLOG.md


В Project Code

AGENT_BOOT.md


Если документ отсутствует — он должен быть создан.


---

3. START SESSION (обязательный этап)

Каждая новая сессия обязана начинаться с этого этапа.

Шаги

1. Агент читает AGENT_BOOT.md


2. Агент читает NOW.md


3. Агент фиксирует текущее состояние репозитория:

текущая ветка

git status

последние 3–5 коммитов



4. Агент формирует план из 3–7 шагов



Без выполнения START SESSION работа запрещена.


---

4. WORK (исполнение)

На этом этапе агент:

вносит изменения в код

запускает тесты и проверки

исправляет ошибки


Ограничения

агент не пишет в Obsidian во время WORK

агент не принимает архитектурных решений без фиксации

вся память фиксируется только в END SESSION



---

5. END SESSION (обязательный этап)

Каждая сессия обязана завершаться синхронизацией.

Шаги

1. Обновить WORKLOG.md

что сделано

ссылки на коммиты



2. Обновить NOW.md

текущее состояние

что готово / не готово

один следующий шаг



3. Если было принято решение — обновить DL.md


4. Сделать коммиты:

в Project Code

в Project Obsidian




Без END SESSION сессия считается незавершённой.


---

6. Формат NOW.md (канон)

NOW.md обязан помещаться в один экран.

# NOW

## Current Goal

## State

## Blockers

## Next Step

Поле Next Step обязано содержать ровно один пункт.


---

7. Запреты

Запрещено:

хранить состояние в голове или чате

смешивать код и мышление

писать рассуждения в README

выполнять работу без обновления NOW.md

завершать сессию без WORKLOG.md



---

8. Ключевой принцип

> Агент не помнит. Агент синхронизируется.



Код — результат. Obsidian — процесс.
