Core Documents Standard v1

Этот документ фиксирует полный перечень обязательных документов для работы в связке: Project Code ↔ Agent CLI ↔ Project Obsidian.

Документы разделены по ролям и назначению. Это канон. Если документа нет — он должен быть создан.


---

1. IDEA.md — ИДЕЯ И АРХИТЕКТУРА

Где: Obsidian (project brain)

Назначение:

архитектурная идея

разделение кода и мышления

роль агента

принципы и философия


Характер:

долговременный

редко меняется



---

2. PROCESS.md — ПРОЦЕСС РАБОТЫ

Где: Obsidian

Назначение:

стандарт работы

алгоритм start / work / end

правила для агента и человека


Обязан содержать:

роли (Code / Obsidian / Agent)

обязательные файлы

start session

end session

запреты



---

3. NOW.md — ТЕКУЩЕЕ СОСТОЯНИЕ

Где: Obsidian

Назначение:

оперативная память проекта

точка входа для агента


Формат:

1 экран

текущая цель

состояние

блокеры

ОДИН следующий шаг



---

4. DL.md — DECISION LOG

Где: Obsidian

Назначение:

фиксировать принятые решения

альтернативы

причины

последствия


Правило: Если решение влияет на архитектуру, процесс или код — оно обязано быть здесь.


---

5. WORKLOG.md — ЖУРНАЛ РАБОТ

Где: Obsidian

Назначение:

хронология действий

что было сделано

ссылки на коммиты


Без:

философии

объяснений

интерпретаций



---

6. AGENTS.md — ТОЧКА ВХОДА АГЕНТА

Где: Project Code repository

Назначение:

быстрый онбординг агента

правила проекта

ссылки на Obsidian

откуда начинать работу


Обязан читаться агентом первым.


---

7. (Опционально) INDEX.md — КАРТА ПРОЕКТА

Где: Obsidian

Назначение:

навигация по идеям, решениям и активным зонам

не дублирует README



---

КЛЮЧЕВОЙ ПРИНЦИП

Код — это результат. Obsidian — это процесс. Agent — это исполнитель, синхронизирующийся с памятью.

Этот набор документов является минимальным стандартом v1.
